Dematting

When PSD compatibility images are flattened and saved out, they are saved composited onto a white background as well as keeping the alpha channel (if there where non-opaque levels in the image).

This, however, creates a problem if you want to use that image to composite onto something else as the white part of the background matte would leak into the alpha channel so to speak, and create a white "halo" around the image.

To deal with this problem, psd-reader uses a de-matte process to remove the original white background from the image, so that the image can be properly composited.

The result of this process is as follows:


Left: Unprocessed original data with background and alpha, Right: de-matting the data produces a proper compositable image with no "halo".

This option is enabled by default as this affects all images with an alpha-channel. psd-reader is smart enough not to invoke any processing if no alpha-channel is present, and the option can be disabled setting the dematte option to false.

For indexed images this is not a problem as there is no alpha channel, just a transparency index if any.

It does not currently work with 32-bit (floating point) color depth images.
